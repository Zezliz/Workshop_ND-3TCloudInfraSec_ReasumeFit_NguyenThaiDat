###  [Blog 1 - SYSTEM MONITORING ON AWS: STREAMING CLOUDWATCH METRICS TO OPENTELEMETRY COLLECTOR IN VPC USING LAMBDA](3.1-Blog1/)
This blog introduces a push-based observability pattern using AWS Lambda and CloudWatch Metric Streams to securely route system performance metrics directly into an isolated VPC-based OpenTelemetry Collector, bypassing traditional API rate limits and avoiding vendor lock-in.

###  [Blog 2 - PRACTICAL EXPERIENCE: BUILDING AN AI GATEWAY TO MANAGE AMAZON BEDROCK USING API GATEWAY](3.2-Blog2/)
This blog shares practical design experience for building a centralized "AI Gateway" using Amazon API Gateway, Lambda Authorizers, and Lambda Integration. This architecture secures LLM access, enforces rate-limiting and quota controls, and ensures a future-proof design for Amazon Bedrock.

###  [Blog 3 - A VERY COOL FEATURE OF AWS LAKE FORMATION](3.3-Blog3/)
This blog explores a new capability in AWS Lake Formation that issues temporary credentials for Spark, allowing direct and secure data access to S3 data lakes based on existing Lake Formation permissions. This solves the common permission mismatch issue between Athena and EMR Spark.
