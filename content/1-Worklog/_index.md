Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Overview of AWS architecture, account creation, and VPC setup](1.1-week1/)

**Week 2:** [Amazon EC2 basics, Security Groups, and SSH access](1.2-week2/)

**Week 3:** [Amazon RDS database setup and EC2 connection testing](1.3-week3/)

**Week 4:** [LAMP stack deployment & WordPress configuration with RDS](1.4-week4/)

**Week 5:** [Amazon Machine Image (AMI) creation & Launch Template setup](1.5-week5/)

**Week 6:** [Application Load Balancer (ALB) deployment & Target Group routing](1.6-week6/)

**Week 7:** [Auto Scaling Group (ASG) integration & Scaling policies](1.7-week7/)

**Week 8:** [Amazon CloudWatch alarms and metric monitoring](1.8-week8/)

**Week 9:** [High Availability testing & ASG CPU stress-test simulation](1.9-week9/)

**Week 10:** [Stateful storage with EFS, CloudFront CDN, and Cloud vs On-Premises comparison](1.10-week10/)

**Week 11:** [Project Design, System Architecture & Workflow Definition](1.11-week11/)

**Week 12:** [Demo Implementation, Testing, System Optimization & Final Report](1.12-week12/)
